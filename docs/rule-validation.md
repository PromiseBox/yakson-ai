# Rule Validation Notes

## Severity 기준

- `RISK`: 병용금기/주의 조합, 연령 금기, 임부 금기/주의처럼 전문가 확인 없이는 보호자가 임의 판단하면 안 되는 항목
- `CAUTION`: 동일 성분 중복, 동일 효능군 중복, 고령자 주의, 수유부 주의, 투여기간 주의, 용량 주의
- `NORMAL`: 분석 대상 약물 중 경고에 포함되지 않은 약물

현재 화면 표기는 `위험`, `주의`, `정상`으로 통일합니다.

## 중복 탐지 정책

- 동일 성분 중복은 같은 `ingredient_id`가 2개 이상 선택 약물에 있을 때 표시합니다.
- 동일 효능군 중복은 같은 효능군에 속하되, 이미 동일 성분 중복으로 설명되는 조합은 제외합니다.
- 같은 약물, 같은 룰 유형, 같은 메시지인 경고는 하나의 카드로 합치고, 근거 ID는 `evidence`에 누적합니다.

## 남은 검증 포인트

- 성분/효능군 마스터 데이터의 임상적 그룹핑 정확도는 약사 검토가 필요합니다.
- `ingredientNames`가 비어 있는 제품은 제품명 괄호 성분으로 검색은 되지만, 성분 테이블 매핑 보강 여부를 별도로 확인해야 합니다.
- 임부/수유부 여부는 현재 성별과 가임 연령 추정으로만 판단합니다. 실제 임신/수유 상태 입력 필드가 생기면 조건을 분리해야 합니다.
- 현재 DB `safety_rule_type` enum에 없는 API 룰 타입은 대표 enum으로 저장하고 원본 타입은 JSON payload의 `apiRuleType`에 보존합니다.
- 현재 결정: 운영 DB enum은 즉시 변경하지 않고, `DUPLICATE_INGREDIENT -> DUPLICATE_EFFICACY` 저장 매핑을 유지합니다. 화면/API 응답은 원본 `DUPLICATE_INGREDIENT`를 그대로 사용합니다.
- 다음 DB 마이그레이션 후보: `analysis_alert.api_rule_type text` 컬럼을 추가해 DB enum 대표 타입과 API 원본 타입을 별도 컬럼으로 분리합니다.
